# Other chain reference 

This document is a summary reference of kind of cryptography used as well as their address format choices. 

| Chain     | Address encoding                                                               | Address                                               | Cryptography used                                                                           | HD wallet approach                                                                                             | Additional comments                                                                                                                      |
|-----------|--------------------------------------------------------------------------------|-------------------------------------------------------|---------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------|
| Cardano   | Bech32                                                                         | Hash of public key(s) / Concatenation of those hashes | EdDSA over Ed25519, blake2b                                                                 | Keys derived with [ED25519-BIP32], with master key derived with [SLIP-0023] and structure following [CIP-1852] | Address is prone to malleability, protection is meant to be achieved by wallets warning users about wrong address set in received tokens |
| Bitcoin   | Base58Check / Bech32 / Bech32m                                                 | Hash of public key                                    | ECDSA/Schnorr over secp256k1, SHA-256/RIPEMPD-160                                           | Keys derived with [BIP-0032] and structure following [BIP-0044]                                                |                                                                                                                                          |
| Ethereum  | Hex / [EIP-55]                                                                 | Last 20 bytes of hash of public key                   | ECDSA over secp256k1, Keccak-256                                                            | Keys derived with [BIP-0032], structure following [BIP-0044] and coin type as in [SLIP-0044]                   |                                                                                                                                          |
| Solana    | Base58                                                                         | Public key                                            | Likely EdDSA over Ed25519, Keccak-256                                                       | Unknown                                                                                                        |                                                                                                                                          |
| Polkadot  | SS58                                                                           | very likely public key                                | EdDSA over Ed25519 and Schnorr/Ristretto over Curve25519, blake2b                           | Mix of [BIP-0032], [ED25519-BIP32], and structure following custom paths approach                              | [Polkadot-accounts], [Polkadot-paths], [Polkadot-cryptography]                                                                           |
| Zcash     | [ZIP-0173] (ZCash-adjusted Bech32m), Base58Check, [ZIP-0316] (Unified Address) | Diversified address or hash of public key             | *Halo2*, *Pallas/Vesta*, *SHA-256*, Blake2, *Pedersen*, *Sinsemilla*, *Poseidon*, *Ed25519* | [ZIP-0032]                                                                                                     | Unified addresses have malleability protection in form of jumbling with Feistel cipher                                                   |
| Monero    | Base58                                                                         | Concatenation of public keys + 4 bytes of its hash    | EdDSA over Ed25519, ring signatures, Keccak-256                                             | Unknown                                                                                                        |                                                                                                                                          |
| Mina      | Base58                                                                         | Compressed public key                                 | Schnorr over Vesta                                                                          | Unknown                                                                                                        |                                                                                                                                          |
| Avalanche | Bech32                                                                         | Last 20 bytes of hash of public key                   | Likely ECDSA over secp256k1, Keccak-256                                                     | Likely keys derived with [BIP-0032] and structure following [BIP-0044]                                         |                                                                                                                                          |
| Sui       | Uknown, likely base16 or base64                                                | Hash of signature scheme byte and relevant public key | EdDSA over Ed25519, ECDSA over secp256k1, ECDSA over secp256r1, Blake2b                     | [SLIP-0010] for Ed25519 curve, [BIP-0032] for secp ones, structure follows [BIP-0044]                          |                                                                                                                                          |
| Aleo      | Bech32 / Base58                                                                | public key derived from a view key                    | Edwards BLS12 / BLS12-377, Poseidon                                                         | Unknown                                                                                                        | [Aleo-curves]                                                                                                                            |

# Address encodings

| Encoding    | Description                                                                                   | Spec                   | Pros                                                                           | Cons                                                                                                                                                                                                                                           |
|-------------|-----------------------------------------------------------------------------------------------|------------------------|--------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Bech32      | Human-friendly encoding with a checksum, avoids using easy to mistype characters like I and l | [BIP-0173], [ZIP-0173] | Popular, checksummed, has human-readable part, used in Cardano                 | [Bech32 issue 51]                                                                                                                                                                                                                              |
| Bech32m     | Iteration of Bech32 with improved checksum                                                    | [BIP-0350]             | Same as for Bech32                                                             | -                                                                                                                                                                                                                                              | 
| Base58      | Alhpanumeric encoding, avoids using easy to mistype characters                                |                        | Popular, double-click just works because address is treated as a single word   | Mixed case makes it harder to type, no human-readable part, relatively slow, unclear error detection capabilities ( Most of the research on error-detecting codes only applies to character-set sizes that are a prime power, which 58 is not) |
| Base58Check | Base58 with a checksum of 4 bytes of SHA-256 hash                                             |                        | Same as for Base58                                                             | Same as for Base58, even slower because of hashing, no error detection guarantees                                                                                                                                                              |
| SS58        | Polkadot's adjustment of Base58Check with network discriminator                               | [Polkadot-accounts]    | Same as for Base58, has some notion of human-readable part, native to Polkadot | Same as for Base58Check, bar the human-readable-part                                                                                                                                                                                           |


Why use hash of public key? 
: As a protection against quantum computers recovering secret keys
